customer/list

애플리케이션의 기능에 따라 URL 경로와 View 파일이 저장되는 폴더를 추가할 수 있습니다.

여기서는 customer 를 경로에 추가하고 View 파일도 customer 폴더에 저장했습니다.

/customer/list 는 Model 인터페이스를 이용해서 데이터(애트리뷰트)를

customer 폴더 list.html 에 전달하는 예시입니다.

서버사이드렌더링에 타임리프 문법을 사용합니다.

애트리뷰트 출력 연습

/customer/list 에서만 전달받는 애트리뷰트 입니다. 보러가기 여기 클릭

  • 애트리뷰트 이름 page 로 저장된 데이터(문자열) 출력
  • 애트리뷰트 이름 dto 로 저장된 객체의 name 필드 출력
  • 애트리뷰트 이름 dto 로 저장된 객체의 address 필드 출력
  • 애트리뷰트 이름 list 로 저장된 list 객체

customer

CustomerMapper(dao) 에서 테이블의 데이터를 모두 조회한 결과조 회 여기 클릭

경로에 하위 폴더가 있을 때에는 / (루트) 사용을 주의하세요.

customer 테이블에서 가져온 애트리뷰트는 값이 없습니다.

/customer/cuslist 에서만 전달받는 애트리뷰트 입니다.

customerId name age email 가입날짜
